# Data Incident Response (v1.0)

## 1) Purpose
Provide a documented process to respond to privacy or data incidents quickly and consistently.

## 2) Scope
Applies to:
- unauthorized access
- accidental sharing
- lost device with sensitive data
- suspected compromise of accounts/tools

## 3) Severity levels (starter)
- Level 1: low impact, quickly contained
- Level 2: moderate impact, requires leadership review
- Level 3: high impact, potential legal/regulatory notifications and full response

## 4) Response steps (public-safe)
1. Contain (stop exposure, disable access, secure accounts)
2. Preserve (keep logs and evidence privately)
3. Assess (what data, who affected, how it happened)
4. Notify (internal leadership; external as required)
5. Remediate (fix root cause, update policies/tools)
6. Document (incident ID + corrective actions)
7. Review (after-action and prevention improvements)

## 5) Documentation
- Public-safe incident log entry may be created without identifiers.
- Full incident report stored privately as RESTRICTED.
