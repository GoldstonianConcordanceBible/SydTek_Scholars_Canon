# Tool Vetting Checklist

Use this checklist before approving any tool used with students or records.

## 1) Purpose & necessity
- [ ] What problem does it solve?
- [ ] Is it required or optional?
- [ ] Is there a safer alternative?

## 2) Privacy & data handling
- [ ] What data is collected?
- [ ] What classification is allowed (PUBLIC/REDACTED/RESTRICTED)?
- [ ] Does the tool store or transmit data externally?
- [ ] Are deletion/export options available?

## 3) Security controls
- [ ] Account controls (unique logins, MFA if possible)
- [ ] Role permissions
- [ ] Audit logs

## 4) Age-appropriateness & safety
- [ ] Appropriate for student age group
- [ ] Moderation controls exist
- [ ] Reporting/blocking controls exist

## 5) Contract/vendor posture (if applicable)
- [ ] Vendor review completed
- [ ] Terms acceptable
- [ ] Data processing terms acceptable

## 6) Approval decision
- [ ] Approved
- [ ] Approved with restrictions
- [ ] Rejected
